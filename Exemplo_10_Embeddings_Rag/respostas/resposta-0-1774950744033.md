No universo do TensorFlow.js, **tensores** são, de forma bem direta, **vetores ou listas em JavaScript cujo conteúdo é representado exclusivamente por números**. Eles funcionam como a estrutura de dados padrão que os algoritmos de machine learning utilizam para identificar padrões, treinar modelos e fazer previsões.

Um ponto fundamental é que o TensorFlow.js **não entende objetos JavaScript ou tipos primitivos** (como strings, booleanos ou estruturas aninhadas) diretamente. Por isso, antes de alimentar um modelo, você precisa "traduzir" seus dados para uma estrutura puramente numérica: o tensor.

Para deixar mais claro, vamos usar o exemplo prático do contexto. Imagine que você tem um array de pessoas com propriedades como nome, idade, cor preferida e localização. Como o modelo não lê texto, convertemos essas informações em uma tabela numérica atribuindo "pontos" (1 para ativo, 0 para inativo) a cada categoria. No caso das cores, por exemplo, a representação ficaria assim:

|        | Azul | Vermelho | Verde |
|--------|------|----------|-------|
| Erick  | 1    | 0        | 0     |
| Ana    | 0    | 1        | 0     |
| Carlos | 0    | 0        | 1     |

Cada linha dessa tabela se torna uma lista de números (um tensor) que o TensorFlow.js consegue processar. Pense no tensor como uma **"planilha numérica"** ou uma **"tradução organizada"** dos seus dados para a linguagem que a inteligência artificial entende.
